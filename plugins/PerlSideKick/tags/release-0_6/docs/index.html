PerlSideKick plugin

(c) 2005 by Martin Raspe

Provides a SideKick structure tree for Perl scripts. I wrote it because I wanted to learn writing jEdit plugins and because the "PerlBrowse" plugin by Will Simpson didn't fully satisfy my needs. PerlSideKick installs itself for the "perl" mode and will display all package names of the current script/module in a tree. The implicit package name "main" will appear on top.

Like all "SideKick" plugins, PerlSideKick has no dockable view. Instead, it shares its view with other SideKick plugins through the SideKick Structure Browser. To use SideKick, dock the Structure Browser on the left or the right. 

Global Options -> Docking -> Structure Browser

In each package branch a use branch is added which lists all used/required modules. Below it follows an alphabetized list of all subroutines. Fully qualified subroutines (e. g. sub Example::Routine) are listed under their corresponding package entries Variables are currently not supported.

The plugin parses the Perl source using regular expressions. An excentric programming style will probably confuse it.

Two options can be set in the Plugin's menu:
Unload sidekick structure browser: Perlsidekick enhances the sidekick structure browser with a Popup menu and marker display. This option replaces the original Sidekick structure browser and takes over all its functionality.
Show Markers: Set it if you want markers to be shown in the Structure Tree (by underlining the relative entry that contains the marker).

Introduction

The main purpose of JavaInsight is to view the content of java bytecode (*.class files) in a disassembled or decompiled form.

JavaInsight comes with two external libraries.

  • The bytecode disassembling is performed by BCEL (Byte Code Engineering Library) by Markus Dahm (markus.dahm@berlin.de).
    More information about BCEL can be found at http://bcel.sourceforge.net.

  • The decompilation is performed by Jode, a tool written by Jochen Hoenicke (jochen@gnu.org).
    More information about Jode can be found at http://jode.sourceforge.net.

Actions

... in Class Tree:

  • Double-click a class to decompile it with Jode.
  • Right-click to show a popup menu with the following actions:

    • Decompile with Jode
      Decompiles the currently selected class with Jode. Only shown, when a class is selected.

    • Add Archives...
      Add a JAR or ZIP file archive to the current classpath.

    • Classpath View
      Switches the tree to classpath view.

    • Package View
      Switches the tree to package view.

    • Flat Packages
      Toggles between flat and hierarchical view of the packages.

... in File System Browser:

Right-click a file. The popup menu shows the following additional actions:

  • Decompile with Jode
    Decompiles the currently selected file with Jode. The file should be a class file, otherwise you'll get an error.

  • Show bytecode
    Show bytecode decompiler output from the BCEL library.

  • Show Jasmin output
    Shows Jasmin assembly output for the currently selected file.

Configuration

To configure JavaInsight, open the Global Options dialog (Utilities->Global Options) and select the entry Plugins->Java Insight.

The following options can be set:

  • Use "pretty" names for local variables.
    Jode tries to prettify the local variable names.
    This option is on per default.

  • Remove locals, that are used only one time.
    Such local variables are defined, but apparently never used. Since they only clutter up the code, they can be removed without problems. Note, that such local variables are only produced by crappy Java compilers; good compilers strip them out anyway.
    This option is off per default.

  • Decrypt encrypted strings.
    This option is meant for obfuscators that "encrypt" strings. Jode tries to recognize encrypted strings and to decrypt them automatically. If activated, Jode searches for expressions like methodX("abc") and assumes that methodX is an decryption routine and "abc" is an encrypted string. It is checked whether methodX is static, final and returns a String. If this is the case, Jode tries to invoke methodX with the constant argument "abc". If successful, Jode replaces the method call with the resulting string.
    This option is on per default, because it shouldn't do anything harmful.

  • Generate imports for packages with more than nnn classes.
    Set this to 0, if you don't want to generate imports for whole packages, such as:
    import java.awt.*;
    The default is 0.

  • Generate imports for classes used more than nnn times.
    Set this to 0, if you don't want to generate imports for classes, rather use full qualified class names always.
    The default is 1.

  • Set status of buffer to "saved" after decompile.
    Use this if you want to get rid of "Save changes?" messages when you close the decompilation buffer.

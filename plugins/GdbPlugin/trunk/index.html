Gdb plugin

Written by Shlomy Reinstein, 4/2007.

Introduction

This plugin enables debugging programs using the gdb debugger directly from jEdit. The plugin provides the following functionality:
  • Setting/Removing/Toggling breakpoints directly in jEdit buffers.
  • Controlling program execution (the usual continue/step/next/...) - either using actions from the GdbPlugin menu or using a very simple control panel.
  • Visualization of breakpoints and the current instruction using icons in the gutter area.
  • Viewing/changing values of expressions and automatic variables (function arguments and variables defined inside the function).
  • Breakpoint dockable which enables editing breakpoint properties (conditions/ skip counts/enabled state) and adding watchpoints.
  • Stack trace (normally called "backtrace" in gdb) dockable that enables you to view and quickly jump to the source code of selected stack frames.
  • Defining "launch configurations" - i.e. the applications to debug, their command-line arguments etc.
  • A console for viewing program output and the gdb output.
The plugin uses gdb/mi (for most functions) to interface with gdb. This is a very incomplete and premature version provided only for preview, it is missing many features and might be quite unstable. Use at your own risk! Despite the above, I found it to be quite helpful in debugging my applications.

Usage

First thing's first: If you don't have 'gdb' in your PATH, you should define the path to gdb using: Plugins->Plugin Options->GdbPlugin->Gdb.

Next, you need to define how to launch your application under gdb. For this, you create a launch configuration using: Plugins->Plugin Options->GdbPlugin->Programs. Click the 'New' button below the "Configurations" box and enter all the details. 'Name' is a symbolic name you assign to the configuration, which simply shows in the "Configurations" list. In 'Environment' you can set a series of environment variables for the debug session in the form var1=value1,var2=value2,...

When you're done filling in the details, click the 'Update' button below them.
The 'Make default' button below the "Configurations" box marks the selected configuration with an asterisk, which means it is now the default configuration, and will be used to launch the next debug session. If the configuration you've just added is not marked with an asterisk, select it in the "Configurations" box and click 'Make default'.

* Note: Some paragraphs will direct you to open dockables. To open a dockable, use Utilities->Global Options->jEdit->Docking then select the appropriate dockable from the list and change its docking from 'floating' to something else and click Ok.

By now, you've set up the gdb path and the launch configuration for debugging. Now, you can start debugging.
You can do the following:

  1. Mark breakpoints in jEdit buffers by putting the caret on the breakpoint line and invoking 'Set breakpoint' from the GdbPlugin plugin menu.
  2. Control the debugger (go!, step, next, quit, pause, ...) using either actions from the GdbPlugin plugin menu or using the dockable named 'Control Panel'.
  3. See the stack trace whenever gdb stops (e.g. in a breakpoint) using the dockable named 'Stack Trace'. When a stack trace is available, you can jump between the frames by clicking on them.
  4. See the automatic variables (arguments + local variables) using the dockable named 'Local Variables'.
  5. Watch the value of custom variables and expressions using the dockable named 'Watches'.
    * The "Variables" dockable provides both (4) and (5) in a split pane, for those who prefer to see both in the same docking area.
  6. Edit breakpoints/add watchpoints using the dockable named "Breakpoints".
  7. Watch the program output using the dockable named "Program Output".
Tips: It is recommended to create keyboard shortcuts or toolbar buttons for quick access to frequent debugging actions. Also, you can use the beta-version of the "Perspective" plugin (which has been merged into the "Docker" plugin in SVN but not released yet) to easily switch between the usual (development) layout and the debugging layout of dockables easily, so you won't have to use the Utilities menu each time to open the debugger dockables.

Feedback (feature requests / bugs / whatever) can be posted using the tracker in SourceForge.

Enjoy!

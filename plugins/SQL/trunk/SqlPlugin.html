SQL plugin

Purpose

The plugin provides different functions for DB developers. It can:
  • Send SQL code to the server. Process results of the publishing in any form: result set, number of rows updated, stored procedure published (with error processing).
  • Browse the DB using standard VFS browser.
  • Download the content of the stored procedures, packages etc.

Configuration

SQL server types

The plugin can work with virtually any type of SQL server (assuming presence of appropriate JDBC driver in ~/.jedit/jars or $JEDIT_HOME/jars or set in CLASSPATH environment variable ).
Each type of the server is given in the XML configuration file. These files stored in ~/.jedit/sql/serverTypes or $JEDIT_HOME/sql/serverTypes (some configuration files - for Oracle, MySQL, PostgreSQL, DB2 are bundled with the plugin). Each file contains the following info:
  • The name of the JDBC driver (optional - if driver is already registered by JVM itself, there is no need to register it in the plugin).
  • Configuration parameters. They are used to generate UI for managing servers and create connection strings.
  • Different service SQL statements.
For an example of SQL server type configuration type, see here .

SQL server records

Each concrete database instance, login, password, it is necessary to create and manage SQL server records. These records are set in "Utilities->Global Options" dialog, "SQL" tree item. The dialog shows the list of the existing server records, allows to create/edit/delete them.

Commands

  • Publish Buffer - Send current buffer entirely to the database. This buffer should contain a correct statement.
  • Publish Buffer To Server - Sends current buffer to the database of user choice (dialog is displayed before publishing). The selection should contain a correct statement.
  • Publish Selection - Sends current selection to the database. The selection should contain a correct statement.
  • Publish Selection To Server - Sends current selection to the database of user choice (dialog is displayed before publishing). The selection should contain a correct statement.
  • Load Object - Downloads the object sources from the database. Creates new buffer. In the drop-down list, red objects are invalid ones, black ones are valid.
  • Load Object From Server - Downloads the object sources from the database of user choice (dialog is displayed before publishing). Creates new buffer.

SqlVFS

The plugin supports the browsing of SQL servers. To access the SqlVFS tree, use the "More->Databases" menu item in the "File System Browser".

Change Log

Version 0.6

  • Name changed to SQL plugin. Thanks to Mike Dillon
  • Greatly improved SqlVFS for Oracle. Separate "directory" for each object type:
    • Tables
    • Views
    • Packages
    • Package Bodies
    • Procedures
    • Functions
    • Triggers
  • Initial support for PostgreSQL (thanks to Pierluigi Mangani) - with partial support of SqlVFS. Patches are welcome.
  • Initial support for DB2, local and remote (thanks to Carmine Lucarelli) - with support of SqlVFS.
  • Some small bugfixes (thanks to user feedback)

Version 0.5

  • OraclePlugin renamed to SqlPlugin.
  • The plugin can work with any type of the SQL server. Each SQL server type is configurable in separate XML file. Loads of internal changes. The plugin does not require oracle JDBC driver to be installed any more.
  • Support of SqlVFS introduced.

Version 0.4

Now the plugin can perform DQL queries to the database. Result set window is added.

Version 0.3

Now the plugin can load objects (procedures, functions etc) from the database.

Version 0.2

Some cleanups. Buffer now can be ended with '/' - this character will be wiped out correctly (I hope).

Version 0.1

Requires jEdit 3.0 (not tested with previous versions). First release I can show to the jEdit community.

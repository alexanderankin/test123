Requirements

  • jEdit >= 2.6pre7
  • plugin EditBus >= 0.9.4
  • plugin Console >= 2.2
  • plugin ErrorList >= 0.11
  • a Java Development Kit (JDK) >= 1.1, not a Java Runtime Environment (JRE) !

Note: the plugins ErrorList and Console are not required directly. JCompiler should start without them. But without them you won't see any error or debug messages.

Installation

  1. Use jEdit's PluginManager to install the JCompiler plugin, or:
    Unzip the contents of JCompiler.zip into the directory <jedithome>/jars:

         $ cd <jedithome>
     $ cd jars
     $ unzip JCompiler.jar

    Depending on your OS and your JDK, here's the location of <jedithome>:

    Operating system JDK Directory <jedithome>
    Windows 95,98,ME SUN JDK, IBM JDK C:\WINDOWS\.jedit
      Microsoft JDK C:\WINDOWS\JAVA\.jedit
    Windows NT all C:\WINNT\profiles\<user>\.jedit
    Unix all $HOME/.jedit
    Macintosh ??? ??? (please tell me)

  2. If you have an JDK 1.1, you're ready.
    If you have an JDK 1.2 or higher (ie. a JDK for Java2), you must perform an additional installation step:

    Make sure, that the file tools.jar from the JDK lib directory is in your classpath. This can be done in two ways:

    1. Copy <jdkdir>/lib/tools.jar to <jedithome>/jars, or
    2. Set the environment variable CLASSPATH to contain tools.jar, ie:
      • on Unix:
                   export $CLASSPATH=$CLASSPATH:/lib/tools.jar
      • on Windows: 
                   set CLASSPATH %CLASSPATH%;\lib\tools.jar

Usage

JCompiler basically has 3 menu actions in the menu (Plugins->JCompiler):

  • Compile File

    Compiles the file currently being edited.


  • Build Package(s)

    Builds the package currently being edited. This is determined by finding out the package that the current file is in and then goes it's root directory and then finds all sub packages. It does a smart compile in that if you have compiled a file and it's .java file hasn't changed then it doesn't recompile it.


  • Rebuild Package(s)

    Does basically what "Build Package(s)" does but always compiles every file.


These menu entries correspond to the following commands, that can be invoked from the JCompiler shell in the Console plugin:
  • compile = Compile File
  • compilepkg = Build Package(s)
  • rebuildpkg = Rebuilds Package(s)

Important Note:

The see the output of the compile process, you must enable the Console and switch to the JConsole shell before you invoke the command.

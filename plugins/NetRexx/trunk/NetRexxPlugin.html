Introduction

The NetRexx plugin adds support to jEdit for interpreting or compiling and running scripts, macros, and programs written in NetRexx. Because the NetRexx language can be compiled into Java classes or directly interpreted, there are two modes available in this plugin - one for compiling and running NetRexx programs and another for interpreting scripts and macros.

Requirements

The minimum (known) requirements are:

  • JRE 1.5 (tested with JRE 1.6.0_35)
  • jEdit 4.3 final
  • Windows or Linux
  • NetRexxC.jar (Rexx Language Association Open NetRexx version 3.02 or later)
  • The Console plugin for jEdit is required to use the Android IDE macros.
Additional plugins needed to activate all features:
  • ErrorList plugin (for error highlighting)
  • CommonControls (for classpath builder function)
  • SideKick (for code structure browser)
The ProjectViewer plugin is recommended for managing NetRexx projects and the SVNplugin is recommended for managing NetRexx source repositories.

Note that this release may work with other Java and jEdit versions or other operating systems but these are the only tested environments at this time. I will adjust these requirements if anyone tests the plugin under different environments and reports success or failure there.

Installation

The 1.0.0 version is available from the jEdit Plugin Central site via the jEdit plugin manager. 

You can also install it manually:
  • Download the NetRexxPlugin.jar file and the NetRexxC.jar file or unzip them from one of the distribution files.
  • Copy both files to the user account's jEdit jars directory or the main jEdit jars directory.
  • Delete any old plugin directories from previous versions:  C:\Documents and Settings\useracct\.jedit\NetRexx and C:\Documents and Settings\useracct\.jedit\macros\NetRexx* before running the new plugin.

    On Windows the user's jEdit jars directory could be C:\Documents and Settings\useracct\.jedit\jars or C:\Documents and Settings\useracct\Appdata\Roaming\jedit\jars (Where "useracct" is the actual userid, of course.)
    The jEdit main jar directory may be C:\Program Files\jEdit\jars
    On Linux the user's jEdit jars directory could be /home/useracct/.jedit/jars

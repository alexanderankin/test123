Introduction

The NetRexx plugin adds support to jEdit for interpreting or compiling and running scripts, macros, and programs written in NetRexx. Because the NetRexx language can be compiled into Java classes or directly interpreted, there are two modes available in this plugin - one for compiling and running NetRexx programs and another for interpreting scripts and macros.

Requirements

The minimum (known) requirements are:

  • JRE 1.5 (tested with JRE 1.6.0_35)
  • jEdit 4.2 final or jEdit 4.3 pre17+
  • Windows or Linux
  • NetRexxC.jar (NetRexx repository tag after3.01-rev264)
Additional plugins recommended to activate all features:
  • ErrorList plugin 1.5 (tested with 1.7)
  • CommonControls
  • SideKick
Note that this is a "beta" release: it is still being tested and it may work with other Java and jEdit versions or other operating systems but these are the only tested environments at this time. I will adjust these requirements if anyone tests the plugin under different environments and reports success or failure there.

Installation

The 0.2.7 version is not yet available from the jEdit Plugin Central site via the jEdit plugin manager. 

You can install it manually:
  • Download the NetRexxScript.jar file and the NetRexxC.jar file or unzip them from one of the distribution files.
  • Copy both files to the user account's jEdit jars directory or the main jEdit jars directory.

    On Windows the user's jEdit jars directory could be C:\Documents and Settings\useracct\.jedit\jars (Where "useracct" is the actual userid, of course.)
    The jEdit main jar directory may be C:\Program Files\jEdit\jars
    On Linux the user's jEdit jars directory could be /home/useracct/.jedit/jars

NetRexxScript test procedure

  1. run propflush and delete jEdit NetRexx directories and jar files to create a fresh test environment
  2. download bin distribution file and unzip in jEdit jars directory
  3. startup jEdit
  4. run all sample scripts with and without console window, with and without trace, preparse/cache, prefix
  5. run all sample scripts from menu and command line
  6. test parse error handling with and without console window
  7. test runcurrent buffer with saved, unsaved and untitled scripts from console and from plugin menu item
  8. check all help pages in jEdit help browser

  9. repeat tests with Ubuntu
  10. repeat tests with jEdit 4.2final
  11. repeat tests with bin+src distribution

About SshPlugin

This is the SshConsole plugin. The plugin, by itself, has no user interface, because it is an extension of three other plugins: Console, FTP, and ErrorList.

Console provides jEdit with a dockable command line interface to the System Shell and the BeanShell, and an open interface for adding new shells. SshConsole adds a jEdit Shell Service to Console called "ssh".

The FTP plugin allows the user to make secure sftp:// connections to remote hosts and browse their file systems, as well as edit their files in jEdit. Since the FTP plugin already asks you for a username/password, and knows what host you are interested in, the SshConsole assumes you want to ssh into the same host.

The ErrorList plugin provides jEdit with a dockable that lets the user click on error messages and have jEdit open the editor at the designated location.

Instructions for use:

  1. Plugins - SshConsole - Show SshConsole [1]
  2. Dock a Utilities - File System Browser and with it, navigate to a remote host via sftp://.
  3. Open a remote file that you want to compile.
  4. If this is your first time connecting, Enter your user/password/key/passphrase there.
  5. After you're connected, if you plan to connect often to this host, add it as a bookmark in the File System Browser.
  6. Enter commands. Compile something. Compile something with Errors.

If later, you use the File System Browser connect to a different remote sftp host, then the next time you use the SshConsole, it should logout of the previous ssh session, create a new shell connection to the same host you just logged into via sftp, and send your shell commands there.

Click the FSB to change directories

A nice feature of this plugin is that you can change the working directory of the remote shell session by clicking on files in the File System Browser. This is also the means by which the ErrorList knows what your current directory is, for the purposes of locating remote files corresponding to errors messages from the ssh console.

This feature is enabled by Console option: Plugin Options - Console - General - On VFS Node Select, cd to node dir

Parse Errors for ErrorList

Console parses the output of the ssh connection through the error parsers which are listed in the Plugin Options - Console - Error Parsers, and creates and adds DefaultErrors to the ErrorSource. You can click on the error in the ErrorList, and jEdit will open the remote file for you if necessary.

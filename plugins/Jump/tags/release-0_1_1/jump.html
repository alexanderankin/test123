Chapter 1. Summary

Jump! work in collaborate with "ProjectViewer" plugin and with "Exuberant Ctags" (ctags.sourceforge.net). It provide several actions which, I hope, accelerate your work with jEdit. First of all, you must indicate path to ctags executable (see Jump! options). Then, when ctags parsed all files of current project, you can move among tags.
Note: you must select the project before Jumpings!

Chapter 2. Usage

Jump to file

Show list of all files founded in current project. You may freely navigate through. Press Enter or Space on list item to jump to desired file. This list also provide incremental navigation, try it. Pressing PageUP, PageDOWN, HOME, END are also valid.

Jump to tag

Show list of all tags founded in current file. Navigation rules as in Jump to file action, plus pressing INSERT (as well as S+CLICK) - will insert picked tag into current caret position.

Search tag

Show list of files where desired tag is founded. First of all, move the carret to desired tag, for example, public final TextAreaPainter get*Painter() (* - caret). When execute "search tag" action you will see list of all files (within current project) where desired tag is founded.

Type a tag...

Displays input dialog to type a tag to search. Press OK to jump to a tag. If multiple occurence of tag founded, it display list of all of them.

Jump Back

Stack type history. After several "search tag" jumpings, you may go back to tags, which you already vistit.

Reload tags for project

Reload the tag file for current project. See Troubleshooting chapter for more info.

Chapter 3. Troubleshooting

If something goes wrong, try to "reload tags for project" action. I hope it help. Also, see Options - Jump, where you can test ctags setup. This test works like: ctags.exe "user.home/.jedit/properties" >> null, just for setup test purpose.
Anyway, mail me if you find some bugs.

Chapter 4. For plugin authors

Jump! stores tag file at non-typical manner. Instead of store ctags output, it serialize CTAGS_Buffer object, which, I hope, is comfortable for any purposes. It save tag file into projectviewer/<projectname>.jump. If you want to use this file, try CTAGS_BG.loadBuffer(String filename).
For details - see sources.

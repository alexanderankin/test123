FTP Plugin By Slava Pestov
sp@gjt.org
The FTP plugin allows you to browse directories and edit files on remote FTP sites. In previous jEdit versions, FTP support was in the editor core; as of jEdit 2.7, it has been moved into a plugin.

To connect to an FTP server, either select Connect to FTP Server from a VFS browser's More menu, or use the Open from FTP Server and Save to FTP Server commands in the Plugins>FTP menu.

jEdit caches remote directory listings to improve performance. Changing files from within jEdit will automatically update the cache as necessary. However, if changes are made from another program, you will need to clear the cache manually using the Clear Remote Directory Cache command in the Plugins>FTP. To force jEdit to forget all passwords, use the Forget Remote Passwords command in the same menu.

By default the FTP plugin uses BINARY mode to transfer files. This will preserve line endings when transfering files between systems with different line separators (e.g. DOS, UN*X, Mac). This option may be disabled in the Ftp options pane in the Plugin Options section of the Global Options pane. When disabled files will be transfered with "Network" line endings ("\r\n" - same as DOS), but most FTP servers will translate these into their native line separator upon upload. So you may use this mode when you would like your saved files to conform to the server's notion of line separators.

Upon download jEdit will leave the files in the Network ("\r\n") mode if BINARY mode is not enabled. If BINARY mode is enabled the files will be opened in jEdit exactly as they appear on the server, with their line endings identical to those in the server copy of the file.

Change Log

Version 0.2.5

  • Provides option to transfer files in binary mode to preserve native line endings. This option is on by default. (Joe Laffey)
  • Fixed a bug introduced in version 0.2.2 that would use ASCII mode by default. (Prior versions correctly used BINARY mode.)
  • File LISTs are done in ASCII mode.

Version 0.2.4

  • No longer uploads/downloads files in ASCII mode

Version 0.2.3

  • No longer tries to CWD into files (Joe Laffey)

Version 0.2.2

  • Made file list parsing code more compatible with Windows and MacOS FTP servers (Joe Laffey)

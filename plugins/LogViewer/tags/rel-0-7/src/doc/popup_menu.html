Log Viewer Plugin

POPUP MENU

The popup menu items should be self-explanatory:
  • Open Log: open a log file.
  • Close Log: close a log file.
    • -
  • Refresh: useful if latency is set to a high number.
  • Reload: same as close then open.
  • Jump to First Line
  • Jump to Last Line
  • Tail: toggles auto scroll on and off for the current log.
  • Find: display a "find" dialog.
    • -
  • Wrap Lines: in table view, wraps cell contents.
    • -
  • Clear Log: just clears the log window, does not alter the log file.
  • Clear All Logs: clears all log windows, does not alter the log files.
  • Delete Log: deletes the current log file from disk.
  • Delete All Logs: deletes all open log files from disk.

Refresh vs. Reload

  • Refresh is useful when the latency period is set to a high value. You may want to see if anything new has been added to the log file before the end of the latency period has been reached. The refresh button will force a read of the log file.
  • Reload is useful when the log file has been deleted or rolled, then recreated, as is often the case with webserver log files. Reload will drop the current file handle and fetch a new one, then reread the entire file.
